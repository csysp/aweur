# Requirements: Awesome Eurorack

**Defined:** 2026-04-19
**Core Value:** A single, well-organized, community-maintained reference that makes it easy to discover open source Eurorack projects by what they do.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Structure

- [ ] **STRC-01**: README.md passes awesome-lint validation
- [ ] **STRC-02**: Table of Contents with links to all sections
- [ ] **STRC-03**: CC0-1.0 LICENSE file in repository root
- [ ] **STRC-04**: Consistent entry format: `[Name](url) \`TAG\` - Description.`
- [ ] **STRC-05**: HW/FW/SW type tags on each entry indicating project type
- [ ] **STRC-06**: Alphabetical ordering within each category
- [ ] **STRC-07**: Introductory paragraph explaining what the list is and how it's organized

### Module Categories

- [ ] **MCAT-01**: Oscillators / Sound Sources category with 2-5 seed entries
- [ ] **MCAT-02**: Filters category with 2-5 seed entries
- [ ] **MCAT-03**: Amplifiers / VCAs category with 2-5 seed entries
- [ ] **MCAT-04**: Envelopes / Function Generators category with 2-5 seed entries
- [ ] **MCAT-05**: Sequencers category with 2-5 seed entries
- [ ] **MCAT-06**: Modulation Sources (LFOs, etc.) category with 2-5 seed entries
- [ ] **MCAT-07**: Effects / Signal Processing category with 2-5 seed entries
- [ ] **MCAT-08**: Utilities (Mixers, Multiples, Attenuators) category with 2-5 seed entries
- [ ] **MCAT-09**: MIDI / USB / Digital Interfaces category with 2-5 seed entries
- [ ] **MCAT-10**: Power Supplies / Cases category with 2-5 seed entries

### Resource Sections

- [ ] **RSRC-01**: Development Tools & Libraries section with seed entries
- [ ] **RSRC-02**: Standards & Specifications section with seed entries
- [ ] **RSRC-03**: Learning Resources / Tutorials section with seed entries
- [ ] **RSRC-04**: Communities & Forums section with seed entries

### Content Quality

- [ ] **QUAL-01**: Every entry links to a real, accessible URL
- [ ] **QUAL-02**: Every entry has a concise, accurate description ending with a period
- [ ] **QUAL-03**: Inclusion criteria defined (what counts as "open source" and "Eurorack")

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Automation

- **AUTO-01**: GitHub Actions CI running awesome-lint on PRs
- **AUTO-02**: GitHub Actions CI running markdownlint on PRs
- **AUTO-03**: Weekly automated link checking (markdown-link-check or lychee)

### Contribution Infrastructure

- **CONT-01**: CONTRIBUTING.md with submission criteria and entry format guide
- **CONT-02**: Pull request template for new entry submissions
- **CONT-03**: Code of conduct

### Enhanced Metadata

- **META-01**: GitHub stars badge per entry
- **META-02**: Last commit / activity badge per entry
- **META-03**: License badge per entry

### Registry

- **REGR-01**: Submission to sindresorhus/awesome registry

## Out of Scope

| Feature | Reason |
|---------|--------|
| Static website or web frontend | Pure Markdown awesome-list — README.md is the product |
| Commercial / closed-source listings | Open source focus only |
| Ratings or reviews of projects | Curation, not evaluation |
| Auto-scraped content | Manual curation ensures quality |
| Video / audio embeds | Markdown-only format |
| Exhaustive metadata tables | Keep entries clean and scannable |
