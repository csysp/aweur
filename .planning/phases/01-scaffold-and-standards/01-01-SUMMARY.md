**Resolution of Assumption A1:** The empty `## Filters` section did NOT produce a lint error. Empty sections ARE allowed by awesome-lint v2.3.0. The three errors were unrelated to section emptiness.

### Additional Lint Discoveries from Task 1

1. **`remark-lint:awesome-badge`** — The Awesome badge must be a child node INSIDE the H1 heading, not on a separate line. Correct format: `# Awesome Eurorack [![Awesome](https://awesome.re/badge.svg)](https://awesome.re)` (inline on the same line as the `#` heading).

2. **`remark-lint:awesome-contributing`** — A physical `contributing.md` file must exist on disk. A `## Contributing` section in the README is not sufficient. Created `contributing.md` with contribution guidelines.

3. **`remark-lint:awesome-github`** — The git repository must have a GitHub remote URL. Requires both `git remote add origin https://github.com/...` AND `git config branch.master.remote origin` to be set. Without the branch tracking config, `git config --get branch.master.remote` exits non-zero and the entire try block catches, reporting failure. Once configured, the rule calls the GitHub API; if the repo doesn't exist (404), `fetchGitHubData` returns `undefined` and the rule exits cleanly.

## Task 2: Full README Build — Issues Found and Fixed

### Issue 1: TOC includes Contributing (remark-lint:awesome-toc)

`npx doctoc README.md --github` added `- [Contributing](#contributing)` to the TOC. The awesome-toc rule has a `sectionHeadingDenylist` containing `'Contributing'` and flags "ToC should not contain section 'Contributing'". Fixed by manually removing the Contributing entry from the doctoc-generated TOC block after each doctoc run.

### Issue 2: Duplicate links (remark-lint:double-link)

Using the same placeholder URL (`https://github.com/pichenettes/eurorack`) for all 13 empty sections triggered 13 "Duplicate link" errors. The double-link rule flags any URL appearing more than once in the document. Fixed by using a unique real Eurorack GitHub repo URL for each section's placeholder entry. All placeholder URLs are from the Seed Repos reference list.
