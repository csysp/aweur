<decisions>
## Implementation Decisions

### Tag Format
- **D-01:** Type tags use backtick code spans: `` `HW` ``, `` `FW` ``, `` `SW` ``
- **D-02:** Full entry format: `[Name](url) \`TAG\` - Description.` (dash separator, period at end)
- **D-03:** Multi-type projects use all applicable tags: e.g., `` `HW` `FW` `` for a project with open hardware and open firmware
- **D-04:** Only three tags: `HW`, `FW`, `SW` — no additional tags (no `DOC`, no `PARTIAL`, etc.)

### Category Ordering
- **D-05:** Module categories follow Eurorack signal flow order: Oscillators → Filters → VCAs → Envelopes/Function Generators → Sequencers → Modulation → Effects → Utilities → MIDI/Interfaces → Power
- **D-06:** Category names are: Oscillators, Filters, VCAs, Envelopes / Function Generators, Sequencers, Modulation, Effects, Utilities, MIDI / Digital Interfaces, Power
- **D-07:** Resource sections are interleaved — not all grouped at the end. Placement TBD during planning (e.g., Dev Tools near module categories, Communities near end)

### Inclusion Criteria
- **D-08:** Open source threshold: any public GitHub repo — no OSI license required, but project must have publicly accessible source files
- **D-09:** Partially open projects: include if hardware files (PCB, schematic) are publicly available, even if firmware or software is proprietary
- **D-10:** Archived/unmaintained repos: include — label with `[Archived]` note in description
- **D-11:** Eurorack scope: Eurorack-primary only — module must be designed specifically for Eurorack (3U, ±12V, 3.5mm jacks). Multi-format projects excluded unless Eurorack is the primary/documented target.

### Intro Paragraph
- **D-12:** Tone is welcoming and descriptive — 3-4 sentences introducing Eurorack for newcomers, explaining the scope, and inviting contributions
- **D-13:** Standard Awesome badge included at the top (links to awesome-list registry)

### Claude's Discretion
- Exact wording of the intro paragraph and inclusion criteria section
- Interleaved placement of resource sections within the TOC (D-07 decided interleaved, exact positions open)
- Whether to include a legend/key explaining the HW/FW/SW tags

</decisions>
